# DiffusionDrive 导出数据说明

## 📊 导出的数据是什么?

### 数据结构
每个 `.pt` 文件包含一个样本,结构如下:

```python
{
    'sample_idx': int,              # 样本索引 (0, 1, 2, ...)
    'token': str,                   # 场景唯一标识符
    'scene_token': str,             # 场景 token
    'planner_outputs': dict,        # DiffusionDrive 模型的完整输出
    'gt_plan': torch.Tensor,        # 真实轨迹 (Ground Truth)
    'ego_fut_cmd': torch.Tensor,    # 驾驶命令 (左转/直行/右转)
    'interface_grid': dict,         # 转换后的 RL 接口数据 (如果启用 --convert)
}
```

### 核心数据详解

#### 1. **planner_outputs** (模型预测)
DiffusionDrive 模型的原始输出,包含:
- **trajectory**: 预测的未来轨迹 (x, y, heading, velocity 等)
- **planning_features**: 中间特征表示
- **其他规划相关的输出**: 如候选轨迹、置信度等

#### 2. **gt_plan** (真实轨迹)
从 NAVSIM 数据集中提取的人类驾驶真实轨迹,作为:
- RL 训练的参考基准
- 奖励计算的依据
- 评估模型性能的标准

#### 3. **ego_fut_cmd** (驾驶命令)
高层驾驶意图:
- `[1, 0, 0]`: 左转
- `[0, 1, 0]`: 直行  
- `[0, 0, 1]`: 右转

#### 4. **interface_grid** (RL 接口数据)
转换后的标准化接口,包含:
- **reference_plan**: 参考轨迹
- **candidate_plans**: 候选轨迹集合
- **plan_confidence**: 轨迹置信度
- **safety_features**: 安全特征

## 🤔 为什么可以用于 RL 训练?

### 传统 RL vs 离线 RL

#### 传统 RL (在线交互)
```
Agent → 环境 → 观察 → 动作 → 奖励 → 更新策略
```
- 需要实时与环境交互
- 在线采样数据
- 适合仿真器/真实环境

#### 离线 RL (从静态数据学习)
```
静态数据集 (观察, 动作, 奖励) → 学习策略
```
- **不需要环境交互**
- 从已收集的数据中学习
- 适合: 人类驾驶数据、专家演示

### 您的场景: **离线 RL (Offline RL)**

```
人类驾驶数据 + DiffusionDrive 预测 
    ↓
离线数据集 (状态, 动作, 参考轨迹)
    ↓
RL 算法 (BC, CQL, IQL, Decision Transformer 等)
    ↓
学习到的策略
```

## 🎯 具体如何使用这些数据训练 RL?

### 方法 1: 行为克隆 (Behavior Cloning, BC)
最简单的离线 RL 方法:

```python
# 伪代码
for sample in dataset:
    state = sample['planner_outputs']['features']
    action = sample['gt_plan']  # 人类专家的轨迹
    
    # 学习模仿人类
    loss = MSE(policy(state), action)
    loss.backward()
```

**原理**: 直接模仿人类驾驶轨迹

### 方法 2: 带奖励的离线 RL
如果您有奖励函数:

```python
for sample in dataset:
    state = extract_state(sample['planner_outputs'])
    expert_action = sample['gt_plan']
    
    # 计算奖励 (例如: 与 GT 的距离、碰撞检测等)
    reward = compute_reward(state, expert_action)
    
    # 使用离线 RL 算法 (CQL, IQL 等)
    loss = cql_loss(policy(state), expert_action, reward)
```

### 方法 3: 基于 DiffusionDrive 的强化学习微调

```python
# 1. 使用 DiffusionDrive 作为初始策略
initial_policy = load_diffusiondrive()

# 2. 从数据集中学习
for sample in dataset:
    state = sample['planner_outputs']
    reference = sample['interface_grid']['reference_plan']
    
    # 生成轨迹
    predicted_traj = initial_policy(state)
    
    # 计算奖励
    reward = evaluate_trajectory(predicted_traj, reference)
    
    # 更新策略
    policy_loss = -reward  # 最大化奖励
    policy_loss.backward()
```

## 💡 为什么不需要实时交互?

### 1. **数据已包含所有必要信息**
- **状态 (State)**: 从 `planner_outputs` 提取
- **动作 (Action)**: `gt_plan` (人类专家的轨迹)
- **上下文**: 传感器数据、地图信息、历史轨迹

### 2. **离线 RL 的优势**
- ✅ 不需要运行仿真器 (快速)
- ✅ 可以复用已有数据 (高效)
- ✅ 不会在探索中产生危险行为 (安全)
- ✅ 可以混合多个数据源 (灵活)

### 3. **适合自动驾驶场景**
- 人类驾驶数据已经是"专家演示"
- 不需要从零开始探索
- 重点是学习安全的驾驶行为

```python
from torch.utils.data import DataLoader

# 1. 加载数据
dataset = DiffusionDriveDataset('data/diffusiondrive_dumps_converted')
dataloader = DataLoader(dataset, batch_size=32, shuffle=True)

# 2. 定义策略网络
policy = YourPolicyNetwork()

# 3. 训练
for epoch in range(num_epochs):
    for batch in dataloader:
        states = batch['planner_outputs']
        actions = batch['gt_plan']
        
        # 行为克隆
        predicted = policy(states)
        loss = F.mse_loss(predicted, actions)
        
        loss.backward()
        optimizer.step()
```

## ❓ 常见问题

### Q: 为什么不直接用 NAVSIM 原始数据?
A: DiffusionDrive 的输出包含了:
- 模型的理解和预测
- 中间特征表示
- 标准化的接口格式
这些比原始数据更适合 RL 训练

### Q: 需要多少数据?
A: 
- 行为克隆: 几百到几千样本即可
- 离线 RL: 越多越好,通常数千到数万
- 您现在的 208 个样本可以开始实验

### Q: 可以混合在线 RL 吗?
A: 可以!典型的流程是:
1. 先用离线数据预训练 (warm start)
2. 再用在线交互微调 (fine-tune)
